# GM 邮件模块 PRD

## 1. 概述

GM 邮件模块用于运营人员向游戏内玩家发送通知邮件或奖励邮件，支持富文本内容、定时发送、条件筛选、道具奖励等功能。

## 2. 功能需求

### 2.1 邮件核心字段

| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | string(128) | 邮件标题，显示在邮件列表 |
| 内容 | text(富文本) | 邮件正文，支持图文混排，使用 **wangEditor**（免费开源） |
| 发送方式 | enum | 立即发送 / 定时发送 |
| 发送时间 | datetime | 定时发送时指定，**使用时区转换存储为UTC**，页面展示按用户本地时区 |
| 目标范围 | enum | 全服 / 指定玩家 / 条件筛选 |
| 玩家列表 | text | 指定玩家时填 **角色ID（playerId）** 列表（每行一个） |
| 有效期 | int(天) | 邮件领取有效期，到期后邮件消失 |
| 条件-最低等级 | int | 仅向 >= 该等级的玩家发送 |
| 条件-最高等级 | int | 仅向 <= 该等级的玩家发送 |
| 条件-最低VIP | int | 仅向 >= 该VIP等级的玩家发送 |
| 条件-最高VIP | int | 仅向 <= 该VIP等级的玩家发送 |
| 状态 | enum | 待审核 / 审核通过 / 已发送 / 发送失败 / 已撤回 |
| 附件奖励 | list<ItemReward> | 邮件附件道具奖励 |
| 道具数量限制 | int | 下限 1，上限 99999999 |

### 2.2 附件奖励（ItemReward 概念模型）

```
ItemReward {
    itemId: Long          # 道具ID（从道具表选择）
    itemName: string      # 道具名称（回显）
    count: int            # 数量
}
```

支持多行道具，通过道具配置表（`t_item_config`）提供快捷选择。

### 2.3 目标范围逻辑

| 目标范围 | 行为 |
|---------|------|
| 全服 | 向该服务器分组下的所有玩家发送 |
| 条件筛选 | 根据等级/VIP条件匹配玩家后发送 |
| 指定玩家 | 向填写的 **角色ID（playerId）** 列表发送（每行一个） |

### 2.4 发送策略

- **立即发送**：审核通过后即时推送
- **定时发送**：审核通过后在指定时间推送
- 定时发送任务使用 **ruoyi-job（XXL-Job）** 模块管理，支持分布式调度
- 所有时间字段存储为 **UTC**，页面展示根据用户本地时区自动转换（支持跨时区场景，如服务器在美国、运营人员在中国）

### 3.2 GM邮件审核页

路径：`/gm/mail/audit`

功能：
- 待审核邮件列表（来源新建/编辑后提交的）
- 审核操作：通过 / 驳回
- 通过时：如果为立即发送 → 直接发送；定时发送 → 写入定时任务
- 驳回时：填写驳回原因，退回待修改状态

## 5. 道具快捷选择组件（通用组件）

**组件名**：`ItemSelector`

**功能**：从项目道具表（`t_item_config`）中搜索并选择道具，支持多选，输出道具列表。

**Props**：
- `projectId`: 项目ID（必填，用于加载对应项目的道具表）
- `value`: v-model 绑定，`[{itemId, itemName, count}]`

**UI**：
- 搜索框：输入道具ID或名称模糊搜索
- 表格/列表：展示匹配的道具，可勾选
- 数量输入：每个选中的道具可输入数量
- 选中列表展示已选的道具及数量

**使用场景**：
- GM 邮件附件奖励配置
- 后续 GM 道具发放功能

## 6. 审批工作流

```
[草稿] ──提交审核──→ [待审核] ──通过──→ [待发送]
                           │                  │
                           ├──驳回──→ [草稿]  │ (立即) → [已发送]
                           │                  │ (定时) → 定时任务 → [已发送]
                           │                  │
                           │                [发送失败]
                           │
                      [已撤回] ←──撤回──
```

- 只有 `待审核` 状态的邮件可以编辑或删除
- 审核通过后不可撤回（已发送的邮件无法撤回）
- 发送失败的需要人工介入处理
