## 二、我的推荐方案（组合）

### 上侧：运营核心指标

四个统计卡片，数据来自 ClickHouse：
| 卡片 | SQL 来源 | 说明 |
|------|---------|------|
| 今日活跃角色 | clickhouse_game_analytics.event_login | 当天 player_id 去重 |
| 今日新增角色 | clickhouse_game_analytics.event_create_role | 当天角色创建数 |
| 今日充值 | 渠道分库 br_pay_order | sum(amount) |
| GM 待办 | 主库 t_gm_mail | status=0 待审核 + failedServerIds 不为空 |

### 中间：待办事项列表

```
┌─ 待办事项 ─────────────────────────────────────────┐
│ ⚠ 待审核 GM 邮件：3 封                    [前往审核] │
│ 🔴 推送失败的邮件：1 封                      [前往重推] │
│ 📋 待配置的道具项目：2 个                    [前往配置] │
└────────────────────────────────────────────────────┘
```

### 下侧：快捷入口（图标式）

```
┌─────────────────────────────────────────────────────┐
│ 📧 GM邮件  │ 👤 角色查询  │ 💰 订单查询  │ 🎁 道具配置  │

```

## 三、实施建议

### 第一阶段（快速见效）
1. 替换首页为欢迎页面 + 快捷入口 + 待办事项
2. 不涉及 ClickHouse，仅用主库数据
3. 1-2天可以完成

### 第二阶段（数据驱动）
1. 接入 ClickHouse 展示实时运营数据
2. 加图表可视化
3. 需要编写 ClickHouse 查询接口

### 关键数据接口（需要新增的 API）
- `GET /game/dashboard/stats` — 今日运营数据概览
- `GET /game/dashboard/todo` — 待办事项统计
- `GET /game/dashboard/recent` — 最近操作记录

## 五、技术实现

### 使用现有组件
- `el-row` / `el-col` 布局
- `el-card` 卡片
- `el-statistic` 或自定义数字展示
- `el-button` 快捷操作
- 图表：可以用 echarts（假如项目有）或简单 CSS 柱状图

### 后端接口复用
- 待办计数：`GmMailMapper.selectList` 加 status 条件 count
- 项目统计：`GameProjectMapper.selectGameProjectList`
- 后续 ClickHouse 数据：`ClickHouseQueryHelper` 已存在复用
